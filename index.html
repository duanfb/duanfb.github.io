记录使用Hexo搭建个人博客

发表于 2017-06-01

注册GitHub账户并新建仓库

1.访问Github官网注册账户https://github.com/
2.新建仓库名为：xxx.github.io

安装Node.js

1. 下载地址:https://nodejs.org/en/download/
2. 依照提示步骤安装即可
3. 安装过程，软件已自动添加环境变量到Path，打开cmd：hexo -v验证是否安装成功

下载Git

https://git-scm.com/并安装，配置环境变量并验证是否配置成功
生成公钥添加到你的github账户

创建本地Hexo仓库

*新建一个目录如：blog，在该目录下打开cmd，
    1.安装Hexo 输入：npm install -g hexo
    2.初始化blog,执行：hexo init
    3.生成静态页面：hexo generate (hexo g也可)
    4.本地启动：hexo server
        浏览器输入http://localhost:4000即可查看

GitHub关联你的本地仓库

1.打开hexo本地仓库blog，找到_config.yml  
     title: Neo博客
     subtitle: 技术博客
     description: 分享学习工作经验等
     author: Neo Duan
     language: zh-Hans
     timezone: Asia/Shanghai

deploy:
  type: git 
  repo: https://github.com/duanfb/duanfb.github.io.git 
  branch: master

  2.按照你的配置修改以上后，执行：npm install hexo-deployer-git --save
  使hexo关联到git

部署本地Hexo到GitHub

1.由于是通过git上传到github，所以前面步骤配置SSH是为了这步考虑
2.部署三步走：
    hexo clean //clean
    hexo g //生成静态页面
    hexo d //发布并push到Github
3.成功后，通过https://XXXX.github.io/即可查看

相关问题

ERROR Deployer not found: git 或者 ERROR Deployer not found: github
解决方法： npm install hexo-deployer-git --save
如果还是报以上错误，建议再次新建一个本地仓库
git上传成功，但是网页未更新，建议重新修改文件多次部署

常用命令

hexo new"postName" #新建文章
hexo new page"pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本
